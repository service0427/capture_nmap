# Naver Map Simulation Project - V1.3.0 Final Report

본 문서는 네이버 지도 앱(v6.4.0.7) FDS 우회 시스템의 최종 사양 및 운영 가이드를 집대성한 문서입니다.

## 1. 하이-피델리티 변조 규격 (Final Spec)

### A. 위치 및 물리 엔진 (v12.2)
- **수평 정확도 (필드 5)**: 4.5m ~ 9.5m 가우시안 노이즈.
- **고도 데이터 (필드 6)**: 15.5m ~ 17.5m (±0.1m 실시간 지터).
- **수직 정확도 (필드 7)**: 3.5m ~ 8.5m 랜덤 주입 (물리적 정당성 확보).
- **방위각 (Bearing)**: 진행 방향 기준 ±0.05도 미세 흔들림.

### B. 환경 및 신분 변조 (v13.3)
- **Identity**: SSAID, IDFV, ADID 전면 랜덤화.
- **Hardware**: 기기별 족보 DB 기반 RAM/Storage/Display/BuildID 세트 주입.
- **Energy**: 배터리 잔량(65~95%), 충전 여부(70%), 온도(28~42도) 시뮬레이션.
- **Uptime**: 부팅 후 경과 시간(0~600초) 랜덤 오프셋 부여.

## 2. FDS 우회 핵심 로직

1. **Standalone Stealth**: MITM 프록시 없이 폰 내부에서 헤더를 세탁하여 탐지 회피.
2. **Partial Reset**: 지도 데이터(Cache)는 보존하되 검색 히스토리와 식별자만 정밀 삭제하여 트래픽 최적화.
3. **Double-Check Sync**: `routeend`의 물리적 시간과 `trafficjam/log`의 통계 수치를 1초 오차 이내로 동기화.

## 3. 운영 가이드 (Best Practices)
- **명령어**: `./start.sh --reset --random --device --ip --gps --nomitm`
- **주의사항**: 장기 주행 시 네트워크 타임아웃(테더링 끊김) 발생 시 기기 재부팅 권장.
- **확인**: 주행 종료 후 로그 폴더의 `trafficjam_log` 패킷에서 필드 11(값:2), 13(주행 시간) 일치 여부 확인.
