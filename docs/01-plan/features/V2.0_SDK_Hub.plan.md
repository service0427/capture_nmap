# [Plan] V2.0_SDK_Hub - Native SDK Hosting Strategy

## 1. 개요 (Objective)
기존의 수동 로직 재현(Emulation) 방식에서 벗어나, 실제 APK 내부의 네이버 지도 SDK를 Java 환경에서 직접 구동(Hosting)하여 100% 데이터 정합성을 보장하는 고성능 패킷 엔진을 구축합니다.

## 2. 핵심 구성 요소

### 2.1. SDK Container (Java)
- **Real SDK Integration**: `navermap-sdk.jar`를 직접 로드하여 내부 상태 머신(State Machine) 활용.
- **Mocked Android API**: SDK 구동에 필수적인 `Context`, `SharedPreferences`, `Handler` 등을 가상화하여 제공.
- **Direct Event Injection**: 실제 UI 클릭 대신 내부 리스너(`onLocationChanged`, `onAction`)를 강제 트리거.

### 2.2. Autonomous Traffic Pipe
- **OkHttp Interceptor**: SDK가 생성한 원본 Protobuf 패킷을 낚아채는 스파이 레이어.
- **Async Push Agent**: 자바 서버가 패킷 완성 즉시 파이썬에게 전송하는 이벤트 구동형 구조.

## 3. 구현 단계

### Step 1: SDK 인프라 구축
- APK에서 추출한 SDK를 프로젝트에 포함시키고, `NTrackerCore` 등 핵심 객체를 메모리에 올리는 단계.

### Step 2: 이벤트 브릿지 설계
- 파이썬의 명령(좌표, 클릭 등)을 자바 내부 SDK 리스너로 전달하는 통로 개설.

### Step 3: 패킷 수확 파이프라인 안정화
- 60초 주기 헬스체크와 주행 로그가 실시간으로 수확되어 서명 서버로 전달되는 흐름 완성.

## 4. 기대 효과
- **Data Symmetry 100%**: SDK 고유의 시퀀스 누적 및 벌크 전송 로직을 그대로 활용하여 탐지 원천 차단.
- **개발 효율성 극대화**: 복잡한 Smali 분석 없이 SDK의 입/출력단만 제어하여 양산 속도 확보.
