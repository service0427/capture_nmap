# [Plan] V2.0_SDK_Hub - Native SDK Hosting & Event Bridge

## 1. 개요 (Objective)
파이썬 재구현 시 발생하는 '데이터 비대칭' 문제를 원천 차단하기 위해, APK 내부의 네이버 지도 SDK를 Java 환경에서 직접 호스팅(Hosting)합니다. 파이썬은 시나리오 관리 및 데이터 가공만 담당하고, 실제 패킷 생성은 진짜 SDK의 상태 머신에 맡겨 100% 정합성을 보장합니다.

## 2. 하이브리드 제어 아키텍처

### 2.1. Native SDK Container (Java)
- **Real SDK Execution**: `navermap-sdk.jar`를 직접 로드하여 `NaviSDK`, `NLog` 엔진 가동.
- **Listener Injection**: 리플렉션을 사용하여 실제 UI 클릭 대신 내부 리스너(예: `onLocationChanged`, `onGuideEvent`)를 강제 트리거.
- **Symmetrical Output**: SDK가 생성한 OkHttp 요청 본문(Protobuf)을 그대로 가로채어 데이터 무결성 유지.

### 2.2. Event Parameter Bridge (Python)
- **Command Dispatcher**: 파이썬에서 `{event: "MOVE_TO", params: {lat, lng}}` 형태의 고수준 명령 하달.
- **Response Processing**: Java Hub로부터 받은 원본 패킷에 `NaverCrypto` 서명을 입혀 최종 전송 및 응답 관리.

## 3. 구현 및 분석 로드맵

### Step 1: SDK 객체 네트워크 구축
- `NTrackerCore`와 `NaviSDK` 간의 의존성을 해결하여 메모리 상에서 주행 세션이 정상적으로 시작되는 상태 구현.

### Step 2: 비동기 자동 반응 로직 (Autonomous Reaction)
- 좌표 주입에 따라 SDK 내부 `GuideEngine`이 도로 상황(카메라, 방지턱)을 스스로 인지하고 관련 로그를 '자동으로' 뱉어내도록 유도.

### Step 3: RPC 인터페이스 안정화
- 파이썬과 자바 허브 간의 명령 전달 지연 시간을 최소화하여 실시간 주행 정밀도 확보.
